H32.2 No Submit Button Warning

Explanation

Success Criterion 3.2.2 (On Input) states that inputting a value into a form field should not result in a change of context unless the user is told that this will happen. The intent is to ensure the user always feels in control with the form, as some may have problems with forms that submit themselves without notice.

Where this is not appropriate, a submit button should be provided for a form. This example does not have a submit button.

Expected Warnings

  • [Standard].Principle3.Guideline3_2.3_2_2.H32.2: "If this form submits automatically on inputting of a field, ensure the user is informed. Otherwise, add a submit button (input type='submit', input type='image', or button type='submit')."
