Success Criterion 2.4.10: Section Headings (Level AAA)

Section headings are used to organize the content.

Note 1: "Heading" is used in its general sense and includes titles and other ways to add a heading to different types of content.
Note 2: This success criterion covers sections within writing, not user interface components. User Interface components are covered under Success Criterion 4.1.2.

(From Understanding WCAG 2.0: Understanding Success Criterion 2.4.10)

Note

No specific test file provided. Where HTML is used, testing for relevant semantic headers (h1-h6) occurs in Success Criterion 1.3.1 - incorrect heading nesting results in an error at AAA level.

Discussion

Summary

The purpose of this Success Criterion is to provide organisation and identification for various parts of a page, using section headings. Headings are important as they facilitate navigation and comprehension, by effectively allowing the user to "skim-read" the content using the headings. Some users will use properly-tagged headings to skip from section to section so they can find the information they want more quickly.

Visual presentation alone (such as horizontal rules) are not considered sufficient, as they do not include the additional mental aids provided by a heading.

Effect on HTML_CodeSniffer

HTML_CodeSniffer does not have any tests that explicitly identify errors under Success Criterion 2.4.10. However, it does affect other tests.

At Success Criterion 1.3.1, there is a test that checks for the order of headings - ensuring that headings are not skipped. For instance, if a h1 element was followed by a h3, a message would appear. At lower levels, it generates a warning, but at Level AAA HTML_CodeSniffer will emit an error. This is because it will not be considered sufficient to satisfy

Note: "This provision is included at Level AAA because it cannot be applied to all types of content and it may not always be possible to insert headings" - for instance, posting pre-existing documents, or something like a letter where headings are not appropriate.
