Success Criterion 2.4.7: Focus Visible (Level AA)

Any keyboard operable user interface has a mode of operation where the keyboard focus indicator is visible.

(From Understanding WCAG 2.0: Understanding Success Criterion 2.4.7)

Discussion

Summary

This Success Criterion is relatively self-explanatory, and is intended to ensure that the user knows where the current focus is within the page, especially for users that intend to navigate through the page using the keyboard.

For standard controls, this will generally be provided by the user agent. It is preferred that this not be interfered with because any changes to the focus indicator at a higher level (such as a High Contrast scheme being applied upon the operating system) will carry through to standard controls. However, for custom-designed controls, this Success Criterion may need to be addressed by the author.

Sniff Coverage

Note that "[Standard]" will be replaced with "WCAG2A", "WCAG2AA" or "WCAG2AAA", depending on the level being tested.

Notices

Message Name Applies To Techniques

Check that there is at least one mode of operation where the keyboard focus indicator can be visually located on user interface controls.
