Success Criterion 3.2.3: Consistent Navigation (Level AA)

Navigational mechanisms that are repeated on multiple Web pages within a set of Web pages occur in the same relative order each time they are repeated, unless a change is initiated by the user.

(From Understanding WCAG 2.0: Understanding Success Criterion 3.2.3)

Discussion

Summary

The purpose of this Success Criterion is to ensure that, normally, elements of navigation are placed in predictable locations within said set of web pages. This should be read in conjunction with Conformance Requirement 3: that a web page that is part of a process cannot conform unless all pages in that process do.

This is relatively self-explanatory on HTML pages - ensure items such as menus, search features, skip-to-navigation/content links, and so on are located in the same position on common pages. However, it extends to other technologies too - two sufficient techniques point to the consistent use of headers and footers, and/or page numbers on PDF documents.
