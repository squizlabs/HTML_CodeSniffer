Success Criterion 2.2.5: Re-authenticating (Level AAA)

When an authenticated session expires, the user can continue the activity without loss of data after re-authenticating.

(From Understanding WCAG 2.0: Understanding Success Criterion 2.2.5)

Discussion

Summary

This Success Criterion requires Web pages wishing to conform at Level AAA, that contain the ability to perform a transaction - such as a form submission or a multi-page process such as a purchase - to provide the ability to save data - preferably automatically - if the process contains the possibility of a user being automatically logged out, and re-construct the transaction in progress following re-authentication.

An automatic logout could exist because of a session timing out, or because the process does not believe the user is no longer who it is. The former is more of a problem, because of issues with users with severe disabilities potentially taking much longer to proceed with transactions.

This Success Criterion, and its sufficient techniques, assume that a form will submit before reporting that re-authentication is required. Some sites attempt to redirect a user to another page after the time-out period for logging back in, using a meta refresh or JavaScript code. This would be a violation of Success Criterion 2.2.1 unless the redirection time-out can be adjusted or removed as described in that Success Criterion.
