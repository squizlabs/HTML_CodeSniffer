Success Criterion 3.3.5: Help (Level AAA)

Context-sensitive help is available.

(From Understanding WCAG 2.0: Understanding Success Criterion 3.3.5)

Discussion

Summary

The purpose of this Success Criterion is to provide help to users to aid in error prevention.

Clear and unambiguous labels can operate as context-sensitive help by themselves. However, other methods of readily-accessible help should be available where further information may be required.

Whether the content author or the user agent provides the relevant help information is deliberately left open. For instance, a user agent that recognises HTML5 may take restrictions set through form validation attributes and turn this into help text.
