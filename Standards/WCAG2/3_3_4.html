Success Criterion 3.3.4: Error Prevention (Legal, Financial, Data) (Level AA)

For Web pages that cause legal commitments or financial transactions for the user to occur, that modify or delete user-controllable data in data storage systems, or that submit user test responses, at least one of the following is true:

  • Reversible: Submissions are reversible.
  • Checked: Data entered by the user is checked for input errors and the user is provided an opportunity to correct them.
  • Confirmed: A mechanism is available for reviewing, confirming, and correcting information before finalizing the submission.

(From Understanding WCAG 2.0: Understanding Success Criterion 3.3.4)

Note

Superseded by Success Criterion 3.3.6 at level AAA, which extends this requirement to all forms.

Summary

The purpose of this Success Criterion is to provide the ability for users to confirm and correct submissions that have legal or other significant ramifications. Users with certain disabilities may make mistakes with entering data - those with motor disabilities may have trouble hitting (or not hitting) certain keys, while other cognitive problems (such as dyslexia) may result in unintended field entries, such as resulting from transposed keys.

What is "user-controllable" data?

The description briefly mentions "user-controllable" data. This recognises that other data may be generated along with the form, such as log data. This Success Criterion is intended to cover data that is entered and can be controlled by the user, such as name and address.

Sufficient methods

The techniques provided as "sufficient" are summarised as the following:

  • Providing a confirmation message before the user finally submits the action, be it an insert, update or delete. On a multi-page form in particular, such a confirmation page should allow a user to review and check their entries before submitting. (G168, G98)
  • Requiring an additional action - such as providing a confirmation checkbox to be checked - before a form can be submitted. This mitigates submitting incorrect or incomplete data based on an accidental click of the Submit button. (G155)
  • Allowing a "cooling-off period" during which an online transaction can be amended or cancelled. Some contracts allow a certain period of time where a commitment can be voided or amended before it is completely binding. (G164)
  • Providing some method to recover deleted items, if no confirmation is provided beforehand - for instance, a Trash Can-style system. An undo mechanism upon delete may also satisfy, so long as the mechanism to undo is accessible. (G99)

If the cooling-off period is used, the method to amend or cancel does not need to be itself available online; it may require a phone call or written notice instead.
