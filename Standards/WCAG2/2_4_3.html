Success Criterion 2.4.3: Focus Order (Level A)

If a Web page can be navigated sequentially and the navigation sequences affect meaning or operation, focusable components receive focus in an order that preserves meaning and operability.

(From Understanding WCAG 2.0: Understanding Success Criterion 2.4.3)

Discussion

Summary

The purpose of the Focus Order Success Criterion is to ensure that sequential navigation of content is performed in an order that preserves the meaning of the content, and can be navigated using the keyboard in a meaningful way.

The main important items to take note of are:

  • Programmatically ordering the fields in a way such that users relying on that order (such as those using a screen reader) can navigate through focusable content in a meaningful manner; and

  • Placing the fields visually in a way so that sighted users using the keyboard to naigate through the page - using the programmatically determined focus order, or some other method - are not confused. This can be ordered by any method that makes mental sense for the content and layout involved.

This is particularly important when content is modified using scripting. If fields are added (such as through the Document Object Model) that see the reading order changed with in a way that interferes with the natural navigation order for a group of users, then this Success Criterion would fail.
